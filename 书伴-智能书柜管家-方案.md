# 书伴 · 智能书柜管家

## 概述

一款面向纸质书读者的Android应用，通过扫描图书ISBN条形码一键录入藏书，利用Google Books API自动获取书籍信息，并结合LLM大模型生成个性化书单推荐，帮助用户轻松管理个人书柜、发现下一本好书。

---

## 核心功能点

| 功能模块 | 说明 |
|---------|------|
| **ISBN扫码录入** | 调用ZXing库扫描图书背面条形码，解析ISBN |
| **书籍信息自动获取** | 通过ISBN调用Google Books API获取书名、作者、封面、出版社、简介等信息 |
| **个人书柜管理** | 书架展示、分类筛选（想读/在读/已读）、书名搜索、删除藏书 |
| **阅读状态追踪** | 标记每本书的阅读状态，记录阅读进度 |
| **AI书单推荐** | 将用户藏书列表发送给LLM，获取个性化推荐；支持输入心情/偏好进行对话式推荐 |
| **AI聊书** | 在书籍详情页调用LLM，以文学评论家视角讨论该书 |
| **读书笔记** | 每本书可添加私人笔记，支持编辑和查看，Room数据库持久化 |
| **主题切换** | 跟随系统 / 浅色 / 深色三种模式 |
| **API Key配置** | 用户在设置页填入自己的LLM API Key（OpenAI兼容格式），本地安全存储 |

## 页面设计

### 1. 启动页 (`SplashActivity`)
- Logo + 应用名称「书伴」
- 1.5秒后自动跳转主页

### 2. 主页 (`MainActivity`)
- **顶部**：欢迎语 + 统计卡片（藏书总数、在读数量、已读数量）
- **中部**：最近添加的书籍（横向RecyclerView，展示封面+书名）
- **底部**：FAB 扫码按钮 → 跳转扫码页
- **底部导航栏**：首页 / 书柜 / 推荐 / 设置

### 3. 扫码页 (`ScanActivity`)
- **扫描按钮**：调用ZXing第三方扫描界面
- **手动输入区**：输入ISBN + 搜索按钮作为兜底
- 扫描成功 → 调用Google Books API → 跳转书籍详情页
- 扫描/搜索失败 → Toast提示

### 4. 书籍详情页 (`BookDetailActivity`)
- **顶部**：书籍封面大图
- **信息区**：书名、作者、出版社、出版日期、页数
- **简介区**：书籍内容简介（Google Books返回）
- **操作区**：
  - 「写笔记」按钮 → 跳转笔记编辑页
  - 「AI聊聊这本书」按钮 → 弹出对话框，LLM返回书评
  - 「加入书柜」按钮（已加入则显示「保存修改」+「删除」）
  - 阅读状态选择器（Chip）：想读 / 在读 / 已读

### 5. 书柜页 (`BookshelfFragment`)
- **顶部Tab筛选**：全部 / 想读 / 在读 / 已读
- **搜索栏**：按书名实时过滤
- **主体**：书籍网格列表（2列GridLayout，显示封面+书名+状态标签）
- 点击某本书 → 跳转详情页
- 长按 → 弹出删除确认对话框

### 6. 推荐页 (`RecommendFragment`)
- **输入区**：用户输入当前心情/偏好的关键词
- **操作按钮**：
  - 「获取推荐」→ 根据输入内容调用LLM
  - 「根据我的书柜推荐」→ 将书柜藏书列表发送给LLM分析口味
- **推荐结果列表**：LLM返回的书名+作者+推荐理由
- 每条推荐结果有「搜索此书」按钮 → 通过Google Books API关键词搜索该书 → 跳转详情页

### 7. 笔记编辑页 (`NoteActivity`)
- **标题栏**：「《书名》· 笔记」
- **编辑区**：多行文本输入框，自动加载已有笔记
- **保存按钮**：存入Room数据库，时间戳记录

### 8. 设置页 (`SettingsFragment`)
- **LLM API配置**：
  - API Endpoint输入框（默认OpenAI格式）
  - API Key输入框（密文显示，带toggle）
  - 模型名称输入（如gpt-4o）
  - 「保存配置」按钮
- **主题设置**：跟随系统 / 浅色模式 / 深色模式（RadioGroup即时切换）
- **底部提示**：API Key仅保存在本地

## 技术实现要点

| 技术点 | 对应功能 |
|-------|---------|
| **Camera + ZXing** | ISBN条形码扫描 |
| **Retrofit2 + Gson** | Google Books API、LLM API的网络请求和JSON解析 |
| **Room数据库** | 藏书信息、阅读状态、笔记的本地持久化 |
| **SharedPreferences** | API Key存储、主题设置 |
| **RecyclerView + GridLayoutManager** | 书柜网格列表、推荐列表 |
| **RecyclerView + LinearLayoutManager(Horizontal)** | 主页最近添加横向列表 |
| **Glide** | 书籍封面图片加载 |
| **TabLayout** | 书柜页的状态筛选Tab |
| **Thread（原生）** | 异步网络请求、数据库操作 |
| **Material Design 3** | 主题样式、日夜间模式切换 |
| **Activity + Fragment 生命周期** | 各页面状态保存与恢复 |

## API调用说明

### Google Books API
- 无需API Key即可基础调用（有限额）
- 接口：`GET https://www.googleapis.com/books/v1/volumes?q=isbn:{ISBN}`
- 返回：书名、作者、封面图URL、出版社、出版日期、页数、简介

### LLM API（OpenAI兼容格式）
- 用户自行填入Endpoint + Key + Model
- 请求格式：
```json
{
  "model": "用户配置的模型名",
  "messages": [
    {"role": "system", "content": "你是一位专业的图书推荐助手..."},
    {"role": "user", "content": "用户输入/藏书列表"}
  ],
  "temperature": 0.7
}
```
- 用于：AI书单推荐、单书讨论（AI聊书）

## 数据模型

```
Book（书籍）：
  id(自增PK) | isbn | title | author | coverUrl | publisher | 
  publishDate | pageCount | description | status(想读/在读/已读) | 
  addedTime

Note（笔记）：
  id(自增PK) | bookId(外键) | content | updateTime
```
